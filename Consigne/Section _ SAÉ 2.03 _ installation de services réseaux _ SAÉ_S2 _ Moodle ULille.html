  • 2ième semaine (semaine calendaire 08)

    -= Format texte à balisage léger =-

     

    Écrire le rapport en Markdown ou AsciiDoc

    Cette semaine, vous devez démarrer l'écriture de votre rapport. Cela commence donc par la description de votre travail de la première semaine et les réponses aux questions.
    Remarque : Le plus important ne sont pas les réponses en elles mêmes, mais la manière de les donner (clarté, lisibilité, précision). Il nous faut les sources qui vont avec vos réponses (les réponses ne doivent pas juste sortir de vos connaissance personnelles mais trouver leur source dans des documents suffisamment sérieux).
     
    • Si vous choisissez le format Markdown + Pandoc (et ses extensions), la documentation de Pandoc est incontournable : https://pandoc.org/MANUAL.html#pandocs-markdown
    • Si vous choisissez AsciiDoc, c'est par ici : https://docs.asciidoctor.org
    Pour ces deux formats, si vous travaillez sur votre machine perso, il y aura des installations à faire. Toutes les informations pour installer ces outils se trouvent dans les documents associées (pandoc et asciidoctor).
     
    Pensez surtout à montrer un maximum d'aspects techniques : gras, italiques, titres, surbrillance…, mais aussi table des matières, listes (différentes sortes), gestion des images (pas d'html dans votre source !), tableaux simples et complexes, les liens internes (par exemple des notes de bas de page, entre le sommaire et les sections) ou externes (par exemple références sitographiques) … La liste n'est pas exhaustive, loin de là.
     
    Faites attention à conserver une cohérence dans l'écriture de votre rapport. On ne met pas des points techniques d'écriture sans que cela s'intègre naturellement dans votre contenu.
     
    Voilà, le plus important est dit concernant l'écriture. 
    Lisez (réellement) le document d'énoncé de la semaine ci-dessous pour avoir plus de détails.

    1- Recherche/analyse applications clientes

    Dernière étape avant de passer à l'outil principal de gestion de sources de projets collaborative. Cette semaine, vous  allez  tester  et  comparer  les
    applications permettant de gérer vos projets avec git via une interface graphique.

    Comme d'habitude, les consignes en pdf et html se trouvent dans le dossier de consigne de la semaine (ci-dessous)

    2- Rendu du rapport intermédiaire

    Déposez une archive contenant votre rapport comme indiqué dans le dépôt ci-dessous. S'il y a un souci de dépôt (modification d'équipe qui ne serait pas prise en compte), n'hésitez pas à nous contacter.

    • Semaine 3 (S09) - Dépôt du rapport technique intermédiaire Devoir
      À rendre : samedi 7 mars 2026, 23:59

      Déposer ici votre rapport intermédiaire (1 dépôt par équipe) sous forme d'une archive zip ou tar.gz (aucun autre format).

      L'archive contiendra :

      1. Un fichier "readme" (en texte simple en balisage léger) indiquant la procédure pour obtenir votre rapport en html et en pdf à partir des sources contenues dans votre archive et toutes informations supplémentaires que vous jugerez utile.
        Souvenez- vous que vous devez simplifier la gestion et la lecture de vos documents. De ce fait, certaines informations ne sont pas explicitement demandées ici mais sont quasiment indispensables. Y penserez-vous ?
      2. Vos sources ainsi que les fichiers supplémentaires lorsqu'ils sont nécessaires à la compilation de votre rapport (images, css, yml …) rangée comme il faut.
      3. Les exports html et pdf de votre rapport.
