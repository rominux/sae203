== Présentation du projet
Ce dépôt contient le rendu final de notre SAÉ 2.03 portant sur l'installation de services réseaux (Debian, Git, Forgejo) et la rédaction de documentation technique en AsciiDoc.

== Structure des documents : Pourquoi deux rapports ?
Suite aux consignes de la Semaine 12, nous avons fait le choix de séparer notre rendu en deux types de documents distincts pour plus de clarté. Nous avons donc deux fichiers maîtres à compiler :

* **Le fichier `tutoriel.adoc` (Le Tutoriel) :** Il raconte notre véritable retour d'expérience sous forme de tutoriel pas-à-pas. Il explique nos choix techniques, les étapes de configuration et les difficultés rencontrées. Chaque semaine y est présentée de manière chronologique, avec des captures d'écran et des explications détaillées pour guider le lecteur à travers notre parcours d'apprentissage.

* **Le fichier `faq.adoc` (Questions / Réponses) :** Il regroupe de manière claire et concise les réponses aux questions théoriques posées dans les différents cahiers des charges, le tout sourcé avec la documentation officielle.

Chacun de ces fichiers maîtres se charge d'inclure (via la directive `include::`) l'ensemble des sous-fichiers correspondants aux différentes semaines.

== Personnalisation du rendu (CSS & PDF)
Pour répondre aux exigences de personnalisation de la dernière semaine, nous avons créé des templates dédiés :

* **Pour le format HTML :** Un fichier `css/style.css` a été créé et lié dans l'en-tête pour moderniser l'affichage (couleurs personnalisées, ombres sur les tableaux, blocs de code type terminal, sommaire repensé). Le rendu HTML et PDF a été testé sur différents navigateurs (Mozilla Firefox et Google Chrome) pour s'assurer de sa bonne compatibilité.
* **Pour le format PDF :** Un fichier `css/mon-theme.yml` a été rédigé pour modifier la police globale, styliser les tableaux et appliquer nos couleurs aux titres via l'attribut `-a pdf-theme` d'Asciidoctor-PDF.

[TIP]
====
**Recommandation de lecture :** Bien que nous fournissions les versions PDF de nos rapports, **nous conseillons vivement de consulter les versions HTML**.
En effet, le style CSS appliqué aux versions web est visuellement plus riche et complet. De plus, la navigation inter-fichiers (les liens permettant de passer d'une semaine du Tutoriel à la FAQ associée) y est pleinement fonctionnelle. Pour des raisons de sécurité liées aux lecteurs PDF modernes bloquant l'ouverture de fichiers locaux, ces liens dynamiques ont dû être supprimés de nos versions PDF.
====

== Procédure de conversion
Notre rapport a été rédigé au format _AsciiDoc_. Pour générer les versions HTML et PDF de nos livrables, nous utilisons le script bash `convertdoc.sh` présent à la racine.

Pour lancer la compilation, exécutez la commande suivante dans le terminal :
[source,bash]
----
./convertdoc.sh
----
Cela générera 4 fichiers nommés proprement (`SAE203_Tutoriel.pdf`, `SAE203_Tutoriel.html`, `SAE203_FAQ.pdf`, `SAE203_FAQ.html`).

=== Outils requis pour la compilation locale
Afin de pouvoir compiler ce document, les outils suivants doivent être installés sur la machine :
* **Ruby** (nécessaire pour l'écosystème Asciidoctor)
* **Asciidoctor** : Convertisseur principal (vers HTML)
* **Asciidoctor-pdf** : Convertisseur vers le format PDF
* **Rouge** : Bibliothèque permettant la coloration syntaxique des blocs de code avec notre thème sombre.

Sur les machines de l'IUT, nous avons installé ces outils localement avec la commande suivante :
`gem install --user-install asciidoctor asciidoctor-pdf rouge`

Nous avons ensuite ajouté le chemin des binaires Ruby au `PATH` de notre environnement utilisateur pour rendre les commandes accessibles.

=== Troubleshooting

Si le script `convertdoc.sh` refuse de s'exécuter avec une erreur du type `Permission denied`, il est fort probable qu'il ait perdu ses droits d'exécution lors du transfert ou du clonage Git. 
Vous pouvez corriger cela très simplement en lui redonnant les droits d'exécution avec la commande suivante :
[source,bash]
----
chmod +x convertdoc.sh
./convertdoc.sh
----
